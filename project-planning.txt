# ##### PROJECT GOAL
- Develop a machine learning system that analyzes transaction data in real-time to identify potential fraudulent activity, reducing financial losses.

# ##### COMPONENTS & SKILLS DEVELOPED
# #### Data Acquisition & Engineering
- use `apache kafka` to simulate a stream of transaction data, or use historical `Kaggle` credit card data to build a SQL-based pipeline.
# #### Exploratory Data Analysis (EDA)
- use `pandas`, `plotly|seaborn|matplotlib` to analyze transactional behavior, location data, and transaction value.
# #### Feature Engineering
- engineer features such as "transaction frequency in the last 24 hours," "distance from home location," and "average transaction amount".
# #### Machine Learning (Imbalance Classification)
- train models (Random Forest, XGBoost) to detect anomalies. handle high-class imbalance using techniques like SMOTE (Synthetic Minority Over-Sampling Technique)
	- or undersampling.
# #### Model Deployment
- deploy the model using `Flask`, `FastAPI`, or `Streamlit`, and containerize with `Docker`.
# #### Monitoring & MLOps
- implement monitoring using tools like `MLflow` or `Airflow` to monitor model degradation over time.

# ##### END-TO-END MACHINE LEARNING PROJECT WORKFLOW
# #### step 1: plan & scope (1-2 days)
- fraud type: online payment/mobile money transactions
- key requirements:
	- real-time scoring: < 100ms latency
	- handle extreme class imbalance: ~0.1%
	- dashboard + alerts
	- production level feel
- success metrics:
	- precision: >= 0.90
	- recall: >= 0.85
	- F1: >= 0.90
	on test set + demo showing live predictions.
- sketch architecture on `draw.io`
- general ML workflow:
	problem definition -> data collection -> data preprocessing -> exploratory data analysis (EDA) -> model selection -> model trainig & validation
		-> model deployment
# #### step 2 grab data (1 day)
- dataset: PaySim (synthetic financial datasets for fraud detection | https://www.kaggle.com/datasets/ealaxi/paysim1
- notebook: https://www.kaggle.com/code/michaelenemona/fraud-detection-system/edit
- download dataset -> open in jupyter -> 1-2 days on EDA (missing values, correlations, fraud patterns by amount/time/type)
- note: transactions which are detected as fraud are cancelled, so for fraud detection these columns (oldbalanceOrig, newbalanceOrig, oldbalanceDest, newbalanceDest)
	- must not be used.
- 1 step is 1 hour of time.
- features: step | type | amount | nameOrig | oldbalanceOrig | newbalanceOrig | nameDest | oldbalanceDest | newbalanceDest | isFraud | isFlaggedFraud
- type: CASH-IN, CASH-OUT, DEBIT, PAYMENT, and TRANSFER
- an illegal attempt in this dataset is an attempt to transfer more than 200.000 in a single transaction.
# #### step 3 set up your tools (30 minutes)
- create a folder and virtual environment
	- create repo on github
	- clone repo to local machine
	- run python `venv` module to create virtual environment: python -m venv env
	- activate virtual environment: env\Scripts\activate
	- install libraries: pip install pandas numpy scikit-learn xgboost fastapi uvicorn joblib sqlalchemy psycopg2 streamlit python-dotenv
	- deactivate virtual environment: deactivate
- the primary purpose of using a virtual environment is to isolate project dependencies, which prevents conflicts between different projects that may require
	- different versions of the same package.
- use `pip freeze > requirements.txt` to save all installed packages and their versions to a file. this allows others to recreate the exact environment on
	- their machines.
# #### step 4 build the ML model offline (3-5 days - data science core)
- in a jupyter notebook:
	- feature engineering: difference in balances, transaction velocity (transactions per step), hour-of-day from step, one-hot on type, etc.
	- handle imbalance: `class_weight='balanced'` in XGBoost or Isolation Forest (unsupervised - great for catching new fraud patterns).
	- train -> evaluate with PR-AUC (better than accuracy for imbalanced data).
	- save model: `joblib.dump(model, "fraud_model.pkl")`.
	aim for 92-96% F1. tons of kaggle notebooks for paysim will give you starter code.
# #### step 5 add real-time inference & end-to-end (1-2 weeks)
- easiest and fastest way to get a working system: clone this exact repo and customize it:
	- https://github.com/uzumstanley/AI-Powered-Real-Time-Fraud-Detection-System | do not clone repo?
	- fastapi backend (instant predictions via api).
	- postgresql storage.
	- isolation forest model + preprocessing pipeline.
	- react? dashboard (live monitoring + alerts) | use tableau dashboard instead
	- quick start in readme - literally runs in minutes.
- for more advanced streaming (if you want kafka on your resume):
	- clone: https://github.com/VeedhiBhanushali/fraud-detection-system
	- kafka producer -> consumer + reddis caching + docker
- simple from-scratch version (if you prefer):
	- create `main.py` with fastapi endpoint that loads your model and predicts.
	- write a `producer.py` script: loop through your test data (or generate fake transactions) and POST to the api every 1-2 seconds.
	- add a streamlit dashboard that polls the api and shows live transactions + red alerts for fraud.
	- store everything in postgres.
	this gives you true "real-time" feel without complexity.
# #### step 6 make it production-ready & deploy (1 week)
	- containerize with docker + docker-compose (include app + DB).
	- deploy free: render.com, railway, or fly.io (one-click).
	- add extras that impress recruiters:
		- SHAP explainability (why this transaction was flagged).
		- simple email/slack alerts.
		- basic drift monitoring (log prediction distribution).
# #### step 7 polish for job applications
	- github README: architecture diagram, metrics, challenges overcome (imbalance, latency), demo video (record screen with live streaming).
	- deployed link + api docs
	- blog post (medium) or linkedin: "I built a real-time fraud detector that scores 10K+ transactions/sec..."
